# The classes involved

Each class participating in the sequence diagram is represented by a box, with a vertical dashed line.

![Classes](Resources/ClassRepresentation.svg)


Sometimes, you wish to clarify the variable, that a method is being called on. So, we can follow the UML convention, and prefix the variable with the class name.

![Variable clarification](Resources/variableRepresentation.svg)

I find that the latter often takes up more space, than is strictly necessary. And while it is more precise, it is rarely needed. So, I usually stick to the former.
