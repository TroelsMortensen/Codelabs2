# Test cases

I have shown different ways to structure your tests in the test classes, pick your favorite.

Regarding test cases, each use case covers more, now, than in the unit testing assignment. There is more to setup, and there is more to verify. You can obviously be inspired by the same test cases, but there is just more work involved.

As seen on the first page, you should cover the use case of selling a stock, and buying a stock. 

Other use cases or features are optional, feel free to add them if you have time.
