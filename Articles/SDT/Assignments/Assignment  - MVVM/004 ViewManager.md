# The ViewManager class

Next, it is good to get the presentation infrastructure in place. That means all those helper classes, which are not views. I suggest starting with the ViewManager class.

Again, you can reuse the class and idea, almost, from previous semester.

You may need to pass data along from one view to another, e.g. some id. You have been shown how to do this, with a `showView()` method that takes a string argument. And an interface for your controllers to implement, to accept the argument.

There is a minor change from last semester, which is a ControllerFactory, versus the ControllerConfigurator you saw. So, you should just read up on that, see link on page 1.
