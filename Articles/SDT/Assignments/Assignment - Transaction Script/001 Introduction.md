# Introduction to the Transaction Script Pattern assignment

In this assignment, you will implement the bulk of your business logic.

* Buying and selling stocks
* Viewing portfolio data
* View other pieces of data
* Managing game state

## Assignment structure

You will focus on the "command" side of things, i.e. updating the data.  
While I _do_ recommend a few query methods, it can be difficult to exactly predict what the presentation layer needs to see, so you may need to add some query methods later.

You are free to design your classes and methods as you see fit. It is up to you to decide how to split the functionality into classes and methods.


## Assignment deliverables

Use cases:
- Buying a stock.
- Selling a stock the player owns.
- Viewing various data relevant to the player's portfolio.
- Managing game state.
- Updated class diagram.
